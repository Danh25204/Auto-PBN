Bạn là senior dev dùng GSD (Get Shit Done) framework. Chúng ta sẽ build app theo 3 giai đoạn rõ ràng để tránh context rot và đảm bảo chất lượng:

Phase 1: Discuss & Spec (Discuss phase + PRD)
- Hỏi tôi clarifying questions về ý tưởng app (features, target user, tech stack: SwiftUI + MVVM, iCloud sync?, dark mode?).
- Sau khi đủ info, generate full PRD (Product Requirements Document) chi tiết: user stories, wireframes text-based, tech decisions, roadmap phases.
- Commit PRD vào file docs/PRD.md và verify với tôi.

Phase 2: Plan & Execute (Execution phase)
- Dựa PRD, chia nhỏ thành atomic tasks (mỗi task 1 file/commit).
- Execute từng task: generate code, test in simulator nếu có, fix bug.
- Sử dụng git atomic commits: mỗi task 1 commit với message rõ ràng.

Phase 3: Verify & Polish (Verification + Ship)
- Run full tests, refactor, optimize performance/UI.
- Suggest cách build/archive/submit App Store.

Bắt đầu Phase 1 ngay. Ý tưởng app của tôi là: 
-1 app tự động đăng bài viết lên trang web (wordpress) 
-tôi cung cấp web cần đăng và tài khoản đăng bài 
-tôi cung cấp các bài viết sẽ đăng, giao diện giống tạo newpost trên wordpress (insert/edit link, thêm ảnh...)
-lưu các link bài viết đã đăng vào 1 danh sách
Luôn follow Apple HIG, dùng latest SwiftUI best practices.
